Statistical Machine Learning · Assignment 8 · Practice · Shorter Tasks, With Solutions

One algorithm, twelve small problems.

Assignment 8 is one idea seen through three lenses. The idea: build a strong model by adding one weak correction per round. Lens one is mechanical — reweight the data so mistakes get louder (AdaBoost, Exercise 1). Lens two is statistical — even after the training error hits zero, more rounds keep growing the margins, so the test error keeps falling (Exercise 2). Lens three is optimizational — the whole procedure is gradient descent in function space, and the base learner class decides which directions the descent can follow (Exercise 3). The problems below are smaller than the sheet's; try each before opening the solution.

Warm-ups · Exercise 1 (and the voluntary Exercise 4)A. The reweighting machine

task 1vote weightcompute

A weak classifier has weighted error ε = 1/5. What vote weight w = ½ ln((1 − ε)/ε) does AdaBoost give it? And if instead ε = 1/2?

Show solution

ε = 1/5: w = ½ ln 4 = ln 2 ≈ 0.69. ε = 1/2: w = ½ ln 1 = 0 — a coin-flip classifier gets no say in the vote at all.

This is exactly the round-1 situation of the sheet's five-point square: the constant rule +1 errs only on the centre (ε₁ = 1/5) and enters the vote with weight ln 2.

task 2the weight updatecompute

Five points with uniform weights 1/5; classifier h₁ (with ε₁ = 1/5, w₁ = ln 2) gets four right and one wrong. Compute the new weights Di(2). What is the total weight of the misclassified points afterwards?

Show solution

Multiply by e±w₁: the wrong point becomes (1/5)·2 = 2/5, each correct point (1/5)·½ = 1/10. Normalizer: 2/5 + 4·(1/10) = 4/5. New weights: wrong point 1/2, correct points 1/8 each.

Misclassified weight afterwards: exactly 1/2 — and that is a theorem, not luck: AdaBoost's choice of wt always reduces the round's own classifier to random guessing on the new weights, which is why the next round must find something new.

task 3the five-point squareexplain

Four positive points at the corners of the unit square, one negative at the centre. Show that every genuine threshold stump has weighted error ≥ 2/5 in round 1 — so the best "stump" is the constant classifier.

Show solution

A stump labels the two sides of one axis-aligned line. Whichever side predicts −1 contains at least two of the four corners (any vertical or horizontal line leaves ≥ 2 corners on each side, and if the −1 side had fewer than two corners it would have to be a thin outer slice — which then misses the centre, so flipping the sign doesn't help either). Two corners at weight 1/5 each give ε ≥ 2/5.

The constant rule +1 errs only on the centre: ε = 1/5 < 2/5. Boosting's first "weak learner" here doesn't split at all.

task 4useless or not?true / false

True or false? "A weak classifier with weighted error εt = 0.9 is useless to AdaBoost."

Show solution

False. Its weight wt = ½ ln(0.1/0.9) < 0 — the vote adds it with a negative coefficient, i.e. effectively uses the flipped classifier −ht, which has error 0.1. A classifier that is reliably wrong is just as informative as one that is reliably right.

The only useless classifiers are the ones with εt = ½ exactly — random-guessing level — which is precisely what the weak-learning assumption εt ≤ ½ − γ rules out.

task 5the stuck caseexplain

Suppose in round t the weak learner returns a classifier with εt = ½ exactly. What happens in round t, and in all later rounds?

Show solution

wt = 0, so the round contributes nothing to the vote — and since the update factors are e0 = 1, the weights don't change either: D(t+1) = D(t). If the weak learner is deterministic it now faces the identical problem and returns the identical answer — AdaBoost repeats the same non-step forever.

That is why the weak-learning assumption demands a fixed margin below ½: not "better than useless once", but bounded away from a coin flip in every round, on every reweighting.

task 6the optimal weightcompute

Adding round t's classifier with an arbitrary weight w multiplies the average exponential loss by Nt(w) = εtew + (1 − εt)e−w. Where is the minimum, and what is its value for εt = 0.15?

Show solution

Setting Nt′(w) = εtew − (1 − εt)e−w = 0 gives e2w = (1 − εt)/εt, i.e. exactly AdaBoost's wt = ½ ln((1 − εt)/εt) (a minimum: Nt″ > 0). Plugging in: Nt(wt) = 2√(εt(1 − εt)).

For εt = 0.15: 2√(0.15 · 0.85) = 2√0.1275 ≈ 0.71 — each such round multiplies the surrogate training loss by 0.71. AdaBoost's weight formula is not a convention; it is the exact line-search minimizer of the exponential loss.

task 7the exponential clockcompute

With edge γ = 0.05 (every round's error ≤ 0.45), the training error after T rounds is at most e−2γ²T. How many rounds guarantee zero training error on n = 10⁶ points?

Show solution

Training error takes values 0, 1/n, 2/n, …, so it is exactly 0 once the bound drops below 1/n: e−2γ²T < 1/n ⟺ T > ln n / 2γ² = 6 ln 10 / 0.005 ≈ 2763.1, so T = 2764 rounds suffice.

Even with a barely-better-than-random learner, the number of rounds grows only logarithmically in the number of training points — a million points cost only a few thousand rounds.

Warm-ups · Exercise 2B. Past zero: margins

task 8boosting past zerotrue / false

True or false? "Once AdaBoost's training error is 0, further rounds can only overfit — the test error cannot keep improving."

Show solution

False — and the sheet's experiment refutes it by a factor of five: test error ≈ 0.022 when the training error first hits 0 (round 7), ≈ 0.004 at round 500. Zero training error only says every margin yif(xi) is positive; further rounds keep growing the margins, pushing the boundary away from the points and into the gap between the classes.

The margin-based generalization bound does not depend on the number of rounds T — larger margins tighten it even when the 0-1 training error can't move.

task 9normalized marginscompute

At round 10 an ensemble has Σws = 10 and a training point has yif(xi) = 2. At round 100, Σws = 60 and the same point has yif(xi) = 27. Raw score up, but did the vote get more confident on this point? Why normalize at all?

Show solution

Normalized margins: 2/10 = 0.20 versus 27/60 = 0.45 — yes, genuinely more confident: the point went from "51%-style majority" toward a decisive vote.

The raw f inflates mechanically because every round adds another wtht with wt > 0 — in the sheet's run max |ft| grows from ≈ 6 to ≈ 266 with no meaning by itself. Only the margin relative to the total vote mass Σws ∈ [−1, 1] compares rounds honestly.

Warm-ups · Exercise 3C. Descent in function space

task 10pseudo-residualscompute

Current predictions f(xi) = (1, 2, 0), targets yi = (1.5, −1, 0.2). Compute the pseudo-residuals for the squared loss ½(y − ŷ)² and for the absolute loss |y − ŷ|. Which loss would you pick if the second point might be a corrupted label?

Show solution

Squared: ri = yi − f(xi) = (0.5, −3, 0.2). Absolute: ri = sign(yi − f(xi)) = (+1, −1, +1).

Under squared loss the suspect point shouts six times louder than anyone else and the next base learner will bend toward it; under absolute loss it is one vote among three. With possible label corruption, the absolute loss is the robust choice.

task 11base learnersexplain

With squared loss, why does gradient boosting with OLS-fitted linear base learners stop making progress after round 1 — and why doesn't the same argument kill tree stumps?

Show solution

Round 1 lands on the OLS prediction Py — the orthogonal projection of y onto the column space S. The new residual y − Py is orthogonal to S (the normal equations), so the best linear fit to it is the zero function: every later round adds nothing. The descent projects onto the same fixed subspace forever.

Stumps don't form a linear subspace: each round may pick a new split and hence a genuinely new direction in ℝn, and a residual orthogonal to one stump is not orthogonal to all of them. The reachable span grows with t, so the training loss keeps falling.

task 12tree depthexplain

Why boost shallow trees rather than fully grown (unbounded-depth) trees?

Show solution

A fully grown tree can put every training point in its own leaf, so it fits the round's pseudo-residuals perfectly — boosting ends after a single round, having simply memorized the labels (the tree version of taking the raw gradient step: training loss 0, nothing learned between the points). Formally, that base class shatters any distinct point set — infinite VC dimension, vacuous generalization bounds.

A shallow tree can only make a small correction per round, so the ensemble's complexity builds up gradually and stops when the loss stops improving — weakness is precisely what makes the base learner boostable.

If you remember one thing: boosting adds one weak correction per round, and each lens of this sheet checks one health condition of that loop. The reweighting keeps every round informative (εt bounded away from ½ — and each round's own classifier is neutralized to exactly ½ afterwards); the margins keep growing after the training error is long zero, which is why stopping at "0 errors" is premature; and the gradient view supplies the design rule — base learners must be rich enough to open new directions each round, weak enough not to swallow the residuals in one gulp.
