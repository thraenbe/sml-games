Statistical Machine Learning · Exercise 3 · Why ℓ¹ Regularization Produces Sparse Solutions

Ridge shrinks every coefficient. The lasso deletes them.

Swap the ridge penalty λ‖w‖² for the ℓ¹ penalty λ‖w‖₁ and something qualitatively new happens: coordinates of the solution become exactly zero. This page walks parts (a)–(e) twice over — once through calculus (a kink in the objective creates a dead zone) and once through geometry (a ball with corners catches projections on the axes).

no penalty — plain least squares LASSO — the ℓ¹ penalty λ‖w‖₁ ridge — the ℓ² penalty ½λ‖w‖²

Part (a) · one dimension is enoughThe kink at zero and the dead zone it creates

Everything about ℓ¹ sparsity is already visible in the simplest possible problem — one weight, one observation:

minimize   ½(w − Y)² + λ|w|

The penalty |w| is smooth everywhere except at w = 0, where it has a kink. Case analysis: for w > 0 the derivative is w − Y + λ, giving the candidate w = Y − λ — but this candidate is only allowed in its own region, so it needs Y > λ. Symmetrically, w = Y + λ needs Y < −λ. And when |Y| ≤ λ, neither candidate is valid: the objective is convex, so the minimum lands on the one point where the derivative doesn't exist — the kink. The result is soft-thresholding:

ŵ = sign(Y) (|Y| − λ)₊   —   exactly 0 whenever |Y| ≤ λ

A whole interval of inputs, the dead zone [−λ, λ], maps to the single output 0. Compare ridge in the same one-dimensional problem (part (c)): minimizing ½(w − Y)² + ½λw² is smooth everywhere, the derivative always exists, and the solution Y/(1 + λ) is zero only when Y is. Slide the thresholds below and watch the difference.

Part (b) · orthonormal designd dimensions = d copies of part (a)

Why does the one-dimensional story matter? Because under an orthonormal design, XTX = Id, the full LASSO problem is d independent copies of it. Completing the square (expand both sides and match terms):

½‖Xw − Y‖² = ½‖w − XTY‖² + ½‖Y‖² − ½‖XTY‖²

The last two terms don't involve w, so the minimizer doesn't see them. What's left — ½‖w − XTY‖² + λ‖w‖₁ — is a sum over coordinates with no cross terms:

Σj [ ½(wj − (XTY)j)² + λ|wj| ]

Each coordinate solves its own part (a), with (XTY)j playing the role of the signal Y. Soft-thresholding applies coordinate by coordinate:

ŵλ,j = sign((XTY)j) (|(XTY)j| − λ)₊   so   ŵλ,j = 0 ⟺ |(XTY)j| ≤ λ

Every feature whose correlation with the data is at most λ is discarded outright. The lasso is a feature selector, and λ is the audition bar.

Part (c) · the ridge contrastSame setup, no kink, no zeros

ridge, coordinate by coordinate

The same completing-the-square reduction applies, so ridge also separates. Each coordinate minimizes ½(wj − zj)² + ½λwj² with zj = (XTY)j. Setting the (everywhere-defined) derivative to zero: ŵridgeλ,j = (XTY)j / (1 + λ).

zero only by coincidence

Since 1 + λ > 0, this is zero iff (XTY)j = 0 exactly — an event of probability zero for noisy data. Ridge divides every coefficient by the same factor: big ones shrink a lot in absolute terms, small ones a little, but none reach zero. The lasso's dead zone |(XTY)j| ≤ λ has positive width; ridge's is a single point.

Part (d) · the geometric reasonCorners catch projections

The penalized problem has a constrained twin: minimize ½‖w − a‖² subject to a budget ‖w‖₁ ≤ t — that is, project the target a onto the ℓ¹ ball. In two dimensions the ℓ¹ ball is a diamond, and its corners sit exactly on the coordinate axes, where one coordinate is zero. Part (d) makes the sparsity claim precise: if a₁ ≥ a₂ ≥ 0 with a₁ + a₂ > t (the target is outside the budget), then

a₁ − a₂ ≥ t   ⟹   ŵ = (t, 0) — the corner

The proof walks three steps. First, since a has nonnegative coordinates, flipping any negative wi to |wi| only helps, so the search lives in the first quadrant. Second, the minimizer must sit on the boundary w₁ + w₂ = t: from any interior point, stepping toward a stays feasible and gets closer. Third, on the boundary the objective is a one-variable quadratic φ(w₁) with unconstrained minimum w₁⋆ = (a₁ − a₂ + t)/2 — and when a₁ − a₂ ≥ t this lies at or beyond t, so the constrained minimum clamps to the endpoint: the corner (t, 0). Strict convexity makes it unique. If the target is lopsided enough, the projection lands on the axis — the second feature is deleted, not merely shrunk.

Part (e) · summaryTwo views of the same phenomenon

penaltysolution (orthonormal X)coordinate is 0 when…geometry
ℓ¹ (lasso) sign((XTY)j)(|(XTY)j| − λ)₊ |(XTY)j| ≤ λ — an interval ball with corners on the axes
ℓ² (ridge) (XTY)j / (1 + λ) (XTY)j = 0 — a single point round ball, no corners

The one-line summary. Sparsity comes from non-smoothness. The kink of |·| at the origin gives the coordinate-wise objective a flat spot of minimizers at 0 — the dead zone — and the same kink, seen as a constraint set, gives the ℓ¹ ball corners that sit on the coordinate axes and catch projections. Ridge's penalty is smooth and its ball is round, so it rescales coefficients without ever selecting among them. If you want a model that chooses its features, you want the penalty with the corner.
