Statistical Machine Learning · Assignment 8 · Exercise 3 · Gradient Boosting

Boosting is gradient descent in function space.

Strip the reweighting story away and boosting becomes something plainer: minimize a loss, one added function at a time, always stepping in the locally steepest downhill direction. The "downhill direction" is a vector of pseudo-residuals — one number per training point — and the base learner's job is to imitate it with an actual function. That framing makes the sheet's real question crisp: which base learners make good legs for the descent? Lines stall after one step; fully grown trees jump straight into memorization; shallow trees are just right.

the current model ft−1 pseudo-residuals — the downhill signal the growing ensemble fit

the algorithm · four stepsStagewise descent

Given data (x₁, y₁), …, (xn, yn), a differentiable loss ℓ(y, ŷ) and a class of base learners hθ, start with f₀ ≡ 0 and repeat:

  1. compute the pseudo-residuals — the negative gradient of the loss with respect to the current predictions;
  2. fit a base learner to them by least squares: θt = argminθ Σ (ri − hθ(xi))²;
  3. pick the step size γt by line search on the actual loss;
  4. update ft = ft−1 + γthθt.

ri = − ∂ℓ(yi, ŷ) / ∂ŷ  |ŷ = ft−1(xi)

part (a) · what the residuals sayTwo losses, two messages to the next learner

squared loss · ℓ = ½(y − ŷ)²

ri = yi − ft−1(xi) — the current residuals. The base learner is trained to predict by how much, and in which direction, the model misses each point. Big miss, big target.

absolute loss · ℓ = |y − ŷ|

ri = sign(yi − ft−1(xi)) ∈ {−1, +1} — only the direction of the miss. Every point, outlier or not, speaks at the same volume: robustness by design.

part (b) · why fit a learner at all?The raw gradient knows nothing between the points

Why not skip step (ii) and update the predictions directly, ft(xi) = ft−1(xi) + ri? Because the gradient (r₁, …, rn) is only defined at the n training points. For the squared loss that update would just write the training labels into the model after one step — training loss zero, and no opinion whatsoever at unseen x. Fitting a base learner means choosing the function in the class {hθ} closest to the negative gradient — an approximate gradient step within a family that is defined everywhere and can generalize. The base class is the channel through which the gradient signal must pass; what survives depends on the class.

parts (c), (d) & (e) · choosing the legsLines stall, deep trees memorize, stumps descend

(c) Linear base learners buy nothing. Take squared loss and OLS-fitted linear base learners. Round 1: the pseudo-residuals are y itself, so the first step lands exactly on the OLS prediction Py (the projection of y onto the column space S, with step size γ₁ = 1 from the line search). Round 2: the new residual y − Py is orthogonal to S — the normal equations from the very first sheet — so the best linear fit to it is the zero function, and nothing ever moves again:

XT(y − Py) = 0  ⟹  ft = f₁ = OLS, for all t ≥ 2

Boosting linear models converges after one round to plain OLS. The failure is structural: every iteration projects onto the same fixed linear subspace, and once the residual is orthogonal to it, the descent is blind. (d) Tree stumps escape because they do not form a linear subspace — each round may pick a different split and thereby open a new direction in ℝn; the residual after one stump is generally not orthogonal to all other stumps, so the loss keeps falling and the reachable span grows with t. (e) Fully grown trees fail the other way: with distinct inputs a deep enough tree puts every point in its own leaf, fits the pseudo-residuals perfectly, and ends boosting after a single round — the tree version of the direct gradient step from (b), memorization included. Shallow trees make a small correction per round, so complexity builds up gradually. Race all three below.

The bigger picture. Gradient boosting reframes "combine weak learners" as plain optimization: the loss defines a landscape over functions, the pseudo-residuals are the local downhill direction, and the base class decides which part of that direction the model can actually follow. That's the design rule the sheet is teaching: pick base learners rich enough that new residuals keep opening new directions (not a fixed linear subspace), yet weak enough that one round can't swallow the residuals whole (not an interpolating tree). AdaBoost itself is this picture with the exponential loss — its wt is exactly the line search of step (iii) — which ties the whole sheet together: the reweighting mechanics are the bookkeeping, and the ever-growing margins are what "keep descending after zero training error" looks like from the outside.
