Statistical Machine Learning · Assignment 8 · Exercise 2 · Implementing AdaBoost

Training error hits zero. Learning doesn't stop.

Run AdaBoost with decision stumps and simply refuse to stop. The training error reaches zero within a handful of rounds — and then, for hundreds of rounds more, the test error keeps falling. By the usual overfitting folklore this shouldn't happen. This page reruns the sheet's experiment live in your browser (a real AdaBoost, trained the moment the page loaded) and shows the quantity that explains the mystery: not whether the training points are right, but how far they sit from the boundary — the margin.

the setupTwo hundred points, an empty band, and stumps

The data: n = 200 training points drawn uniformly from [−1, 1]², but conditioned on |x₁ + x₂| > 0.3 — so an empty diagonal band separates the classes — and labelled by the diagonal, y = sign(x₁ + x₂). The weak learner is an exhaustive decision-stump search: every feature, every threshold between consecutive values, both signs, pick the smallest weighted error. In the sheet's reference run ε₁ = 0.15 and w₁ ≈ 0.87, the training error is 0 from round t₀ = 7, and the weighted errors εt settle around 0.26 — bounded away from ½, so the weak-learning assumption holds with an empirical edge γ̂ ≈ 0.2 the whole way.

Below, the same experiment with this page's own seed: the 0-1 error curves of sign(ft) on train and test, and the decision regions at round t. Start at t₀, where the training error first touches zero — then slide on and watch what the extra rounds buy.

part (b) · the phenomenonFive-fold better, all of it after “done”

In the reference run the test error is ≈ 0.022 at t₀ = 7 and ≈ 0.004 at T = 500 — a more-than-five-fold improvement, every bit of it earned after the training error was already zero. The improvement keeps arriving in steps until roughly round 300. So the training error tells only a small part of the story: boosting keeps learning long after it has interpolated the training labels. The classical picture — “zero training error means the model has stopped extracting signal and started memorizing” — simply fails here, and the margin theory below says why.

part (c) · the explanationMargins: the boundary keeps moving away

For a training point, being classified correctly is a yes/no fact — but yift(xi) measures something finer: how confidently the vote gets it right. Because the raw values grow with every added stump (in the reference run max |ft| inflates from ≈ 6 to ≈ 266), the meaningful quantity is the normalized margin

margini = yift(xi) / Σs≤t ws  ∈  [−1, 1]

Zero training error only says every margin is positive. Further rounds keep increasing the margins — pushing the decision boundary off the points and into the empty band. That is exactly what the margin-based generalization bound from the lecture rewards: P(yf(x) ≤ 0) ≲ P̂(margin ≤ τ) + O((1/τ)√(d log n / n)) — a bound that does not depend on the number of rounds T. Growing margins let the first term shrink for larger τ, so the guarantee (and, as the run shows, the actual test error) can keep improving even though the training error has long been zero.

The bigger picture. “Zero training error” is a statement about signs; generalization here is a statement about distances. AdaBoost minimizes an exponential loss that never saturates — a correctly classified point still contributes e−yif(xi) — so the algorithm keeps paying attention to barely-right points and keeps pushing the boundary away from them, into the gap between the classes. The margin bound turns that into a guarantee that doesn't count rounds: what matters is not how big the ensemble is, but how confidently it separates. Interpolation is not the finish line; it is where margin maximization starts.
