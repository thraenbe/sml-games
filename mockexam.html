Question 1 · 2 points · ≈5 min · easyThe perceptron algorithm by hand

Rules of the game: no offset, start at w = (0, 0), sweep the points in the given order, and repeat until a full sweep makes no update. A point counts as a mistake when yi⟨w, xi⟩ ≤ 0 — note the ≤: the very first point is always a mistake, because ⟨0, x₁⟩ = 0. On a mistake, the update is

w ← w + yixi

— add the point if it is positive, subtract it if it is negative. That drags the boundary toward classifying the offender correctly. The whole run is six rows of arithmetic: two mistakes in sweep 1 (x₁ then x₂), then a completely clean pass, so the algorithm stops with w = (2, 1) after 2 mistakes. Step through it:

rubric · 0.5 + 0.5 for the running weights (1,2) and (2,1) · 1 for marking x₃ and all of sweep 2 as clean, so the run stops

Exam habit worth stealing: write ⟨w, xi⟩ first, then multiply by yi. The classic slip is on x₂ in sweep 2: ⟨(2,1), (−1,1)⟩ = −1 looks like a mistake, but y₂ = −1 makes y₂⟨w, x₂⟩ = +1 > 0 — correctly classified.

Question 2 · 4 points · ≈10 min · mediumMultiple choice: consistency & generalization

Four items, exactly one correct answer each, no penalty for wrong answers — so the only losing move is a blank. The four answers are A, B, D, A:

(1) Consistency → A

Consistency is convergence of the risk: R(fn) → R*. It does not require fn(x) → f*(x) pointwise (B is a stronger, different statement). ERM over a fixed class converges to best-in-class, not to Bayes (C). And no free lunch: there is no distribution-free rate (D).

(2) Universal consistency → B

Stone's theorem: k-NN with k(n) → ∞ and k(n)/n → 0 is universally consistent. Fixed k = 1 is not — the neighbour noise never averages out (A). That same rule refutes “no algorithm can” (C). And universal consistency is asymptotic: for any fixed n, some distribution keeps you far from Bayes (D).

(3) Two bounds → D

Read rates off the formulas: ℰA ∼ √(d ln n / n) decays like 1/√n and grows like √d; ℰB ∼ d ln n / n decays like 1/n — faster in n — but grows linearly in d. “Faster” is not “always smaller”: the figure below shows the constants fighting back.

(4) Sauer–Shelah → A

For n ≥ d the growth function is bounded by Σi≤d (ni) = O(nd) — polynomial, which is exactly the exponential-worry (B) it rules out. Infinite VC dimension means 2n labelings forever (C), and nothing requires ℱ to be finite (D).

rubric · 1 point per item, no penalty · answers: (1) A · (2) B · (3) D · (4) A

Question 3 · 5 points · ≈12 min · mediumCost-sensitive Bayes classification

Setup: prior ℙ(Y=1) = 2/5, class-conditionals ℙ(X=1 | Y=1) = 4/5 and ℙ(X=1 | Y=0) = 1/5. (a) is pure bookkeeping — build the joint table by multiplying prior × conditional: ℙ(1,1) = 8/25, ℙ(0,1) = 2/25, ℙ(1,0) = 3/25, ℙ(0,0) = 12/25. Sum columns for the marginal, divide for the posterior:

μ(1) = 11/25,   μ(0) = 14/25,   η(1) = 8/11,   η(0) = 1/7

The 0–1 Bayes classifier thresholds η at ½: f°(1) = 1, f°(0) = 0, and its error collects the losing posterior mass at each x: Σx μ(x) min(η, 1−η) = 11/25 · 3/11 + 14/25 · 1/7 = 1/5.

(b) Asymmetric loss — a missed positive costs a, a false alarm costs 1. Compare the two conditional risks: predicting 1 risks (1−η)·1, predicting 0 risks η·a. Predict 1 whenever that is the cheaper side:

predict 1  ⟺  1 − η(x) < a η(x)  ⟺  η(x) > 1/(1 + a)

The threshold slides down from ½ as misses get pricier — with a = 1 it is the familiar ½ again. (c) keeps 0–1 loss but turns the prior π into a dial: with the class-conditionals fixed, η(0) = π / (4 − 3π), and the decision at x = 0 flips to 1 (ties toward 1) when η(0) ≥ ½ ⟺ π ≥ 4(1−π) ⟺ π ≥ 4/5. All three parts are the same picture — two posterior dots and one threshold:

Question 4 · 4 points · ≈10 min · transferROC curves and fairness

Equalized odds asks for the same (FPR, TPR) operating point in both groups. (a) is a trap dressed as a gift: the two groups' ROC curves coincide, so surely one global threshold works? Not necessarily. A ROC curve fixes only the set of reachable points — not which threshold lands where. If group 1's scores are group 0's divided by two, the curves are identical but a single global threshold puts the groups at different points of the shared curve, and equalized odds fails. (It is guaranteed only when the per-label score distributions themselves coincide.)

(b) asks for the general fix. Draw the A=1 curve strictly below A=0: each group can realize any point on its curve with a deterministic threshold, and any point below it by randomizing between two thresholds. So the feasible common points are the region between the diagonal and the lower curve. Put the operating point on the A=1 curve itself: the harder group sits there deterministically, while A=0 must randomize — deliberately blunting its better classifier. That is the price of equalized odds, paid in accuracy on the easier group.

rubric · (a) 0.5 verdict “not necessarily” + 1 reason (curves fix the set of points, not the threshold-to-point map) · (b) 1 valid strictly-lower curve + 1 common feasible point + 0.5 noting A=0 must randomize between two thresholds

Question 5 · 4 points · ≈9 min · mediumSpiky-smooth interpolation in high dimensions

The given tool is sphere concentration: two independent uniform unit directions in ℝd are nearly orthogonal, ℙ(|⟨u, v⟩| > t) ≤ 2e−cdt². (a) One test direction against n training directions is a union bound — multiply by n:

ℙ(∃ i ≤ n : |⟨u, vi⟩| > t) ≤ 2n e−cdt²

(b) When does that die? Take logs: ln(2n) − cdt² → −∞, which holds whenever d ≫ log n / (ct²) — any growth with d / log n → ∞ suffices. Dimension only needs to beat the logarithm of the sample size. (c) The geometric punchline: with high probability a fresh test point is nearly orthogonal to every training direction at once. The interpolator can afford to be “spiky” — tall, narrow bumps that nail each noisy training label — because those spikes have vanishing volume. A typical test point misses all of them and lands on the smooth part, where the function is ≈ the truth. Zero training error and small test risk, simultaneously.

rubric · (a) 1 for the union bound · (b) 1.5 for any correct sufficient condition (d/log n → ∞) · (c) 1.5: near-orthogonality + vanishing spike volume + “test prediction ≈ smooth part ≈ truth”

This is Sheet 10's double-descent story in proof form — the same “benign overfitting” regime the min-norm interpolator lives in on the matrix page's formulas.

Question 6 · 5 points · ≈12 min · mediumAlgorithmic stability of the averaging rule

The rule ignores x entirely: A(S)(x) = ȳ, the label average, with labels in [0, 1] and squared loss. Swap example i for an independent copy and follow the damage through three steps:

(a) |A(S) − A(Si)| = (1/n)|Yi − Yi′| ≤ 1/n

— only one of the n summands changed, and by at most 1. (b) pushes that through the squared loss with the difference-of-squares factorization (a sheet-5 classic):

|(ȳ − y)² − (ȳ′ − y)²| = |ȳ − ȳ′| · |ȳ + ȳ′ − 2y| ≤ (1/n) · 2 = 2/n

The first factor is (a); the second is at most 2 because every quantity lives in [0, 1]. Hence Δsup ≤ 2/n. (c) is the payoff via the stability–generalization lemma: the expected gap between true and empirical risk equals the average stability, and |Δav| ≤ Δsup, so

| 𝔼[R(A(S)) − R̂n(A(S))] | ≤ 2/n → 0

What it does say

The training error of the averaging rule tracks its true risk to within 2/n in expectation. The rule cannot overfit: nothing it shows you in-sample is an illusion.

What it does not say

That the risk is small. A constant predictor has huge approximation error whenever the regression function actually varies. Stability kills the estimation gap, not the bias — the exam explicitly rewards saying so.

rubric · (a) 1 · (b) 1 factoring + 1 bounding the two factors · (c) 1 invoking the lemma with |Δ_av| ≤ Δ_sup, 1 concluding the ≤ 2/n gap

Question 7 · 4 points · ≈9 min · mediumAdaBoost and the training-error bound

Two steps, each one inequality long. (a) A point is misclassified by sign(fT) exactly when its margin z = yifT(xi) is ≤ 0, and the indicator of that event is dominated by the exponential: 𝟙{z ≤ 0} ≤ e−z. Average over the points and the right side is exactly the potential:

R̂n(sign fT) ≤ (1/n) Σi e−yifT(xi) = ZT

(b) Since Z₀ = 1, telescope the potential: ZT = ∏t (Zt/Zt−1) = ∏t 2√(εt(1−εt)). With an edge, εt ≤ ½ − γ, each factor is 2√(ε(1−ε)) = √(1 − 4γt²) ≤ √(1 − 4γ²) ≤ e−2γ² (using 1 + u ≤ eu), so T rounds multiply to R̂n ≤ e−2γ²T.

rubric · (a) 1.5 for the 𝟙 ≤ exp step · (b) 2.5 for telescoping Z_T = ∏ 2√(ε(1−ε)) and √(1−4γ²) ≤ e^(−2γ²)

Question 8 · 6 points · ≈14 min · hardVC dimension of interval classifiers

ℱ₁ = single closed intervals, ℱ₂ = unions of at most two. The key reduction: on a fixed ordered point set, an interval classifier can only mark a contiguous block positive — so counting labelings is counting blocks. (a) The empty labeling (1 way) plus a block for every pair of endpoints i ≤ j:

𝒩(ℱ₁, n) = 1 + n + ½n(n−1)   ⟹   VC(ℱ₁) = 2

(For n = 2: 4 = 2² — shattered; for n = 3: 7 < 8 — the labeling +,−,+ is unreachable.) (b) With two intervals the currency is positive runs: maximal consecutive blocks of +. Any labeling of 4 ordered points has at most 2 positive runs — the worst case is +,−,+,− — and two intervals cover two runs, so all 2⁴ = 16 labelings are realizable: VC(ℱ₂) ≥ 4. (c) On any 5 points, the alternating labeling +,−,+,−,+ has three separated positive runs, which no union of two intervals can produce. So no 5-set is shattered and VC(ℱ₂) = 4. Try to beat it:

rubric · (a) 1 counting argument + 1 for VC(ℱ₁) = 2 · (b) 2 for a valid 4-point shattering via the ≤ 2-runs argument · (c) 2 for the alternating 5-point labeling
